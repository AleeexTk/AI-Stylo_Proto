# Product Drafts Summary

This document consolidates and cleans up early brainstorm notes previously stored as loose `.txt` drafts in the repo root.

## 1) RPG-style Streamlit UX concept

Core idea:
- Keep AI recommendation logic (catalog + embeddings + outfit generation),
- Rebuild UX as an RPG inventory metaphor,
- Track profile, slots, wishlist, and event history in `session_state`.

Implementation emphasis:
- onboarding profile (style, budget, size),
- outfit slots (`top`, `bottom`, `shoes`, `accessory`),
- interaction telemetry for personalization loops.

## 2) Must-have prototype scope (business-first)

Primary demo objective:
- show measurable conversion impact, not just “AI wow”.

Must-have features:
1. Product-card block with “pick outfit” and “try on”.
2. Full-look assembly with one-click buy-all CTA.
3. Budget presets (mass / middle / premium ranges).
4. Recommendation rationale under each item.
5. Mobile-first flow.

## 3) Gamification & analytics layer

Proposed lightweight metrics over existing profile/events:
- style progress and skill badges,
- luxury index,
- deal hunter score,
- savings vs splurge indicators.

Goal:
- increase engagement while producing interpretable behavioral signals.

## 4) Trend synthesis (2025–2026)

Key strategic hypotheses:
- outfit-level shopping over single-SKU upsell,
- conversational commerce + personalized AI selection,
- returns reduction via fit/try-on support,
- UA market requires strong budget sensitivity + mobile UX.

## 5) GTM focus (UA)

Commercial framing:
- AI layer for e-commerce stores that improves conversion and basket size,
- pilot around product-card outfit recommendations and bundle checkout.

Priority pilot segments:
- women’s fashion (mass/middle),
- streetwear,
- digitally active local D2C brands.
